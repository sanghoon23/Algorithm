// * 문제

n! = n * (n - 1) * (n - 2) * ... * 3 * 2 * 1;
1 <= n <= 100 (범위)

만약 input 값에 5가 주어지면,
5 * 4 * 3 * 2 * 1 = ?? 의 값을 구하는 수열 문제.
하지만,
n > 20 이 되면 우리가 사용할 수 있는 int 의 데이터 범위를 초과하게 된다.
자바, 파이썬 등에선 제일 큰 범위의 자료형 long long 를 써서 해결할 수 있겠지만,
이 문제는 자료형보다 큰 데이터를 어떻게 우리가 다룰 수 있을지에 관한 문제인 것 같다.

/////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
// * 해결방법

n 의 자리수 마다 값을 구한다. 범위는 주어졌으니 1, 10, 100 의 자리수의 값만 구하면 된다.
그런 다음,
1의 자리수를 곱해서 vector[i] + 해주고
10 의 자리수와 원본 데이터의 값을 곱해서 vector[i + 1] 자리에 += 해준다.
100 의 자리수는 vector[i + 2] +=

마지막으로, 해당 배열의 10 넘는 값들은 다시 value / 10, value % 10 으로 값을 앞자리에 넘겨준다.

/////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
// * 시행착오

1. HEAP CORRUPTION DETECTED 오류.
- new 로 할당한 메모리 영역보다 더 큰 영역에 접근하고자 했기 때문이다.
값을 계속 더하다보면 어느 순간 값이 엄청나게 커지는데, 이 때 size = 100 으로 해놓아서 값이 초과해버린듯 하다.
그래서 size = 10000;

2. 이중포인터
- 해당 값을 집어넣을 배열을 선언하고 그 배열을 이중포인터로 넘겨서, 원본 데이터에서 바로 수정할 수 있도록 함수를 만들었는데,
	이 때, 문제 발생.
	
	CalcVec[i + j] += ((*v)[i] * checkArray[j]); // OK
	CalcVec[i + j] += (*v[i] * checkArray[j]); // ERROR
	CalcVec[i + j] += (*v[i]); // ERROR
	
	(*v) v를 먼저 포인터 계산하고, 그거에 해당하는 배열을 찾아야 되는데
	v[i] 연산이 먼저 되어버린 것이다. 그래서 nullptr 에러.
