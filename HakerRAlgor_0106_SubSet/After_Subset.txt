/////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
// * 문제

Array S = [19, 10, 12, 10, 24, 25, 22] 가 주어지고 K = 4 가 주어진다.
S 에 있는 요소 두 개를 더해서, K 와 나누어 떨어지지 않는 S 의 부분집합의 최대 크기를 구하라.

위 예시에선,
S'[0] = [10, 12, 25]
S'[1] = [19, 22, 24] 이고,

10 + 12 = 22, 22 % k != 0
12 + 25 = 37, 37 % k != 0
10 + 25 = 35, 35 % k != 0
... 이므로,

최대의 부분집합 갯수는 3 이 된다.

/////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
// * 해결 방법

- 목적 : 두 개의 요소를 더해서 K 한테 나누어떨어지지 않는 "최대" 부분집합의 개수 구하기.

우선 예시를 보자.
k = 4 이고,
S = [19, 10, 12, 10, 24, 25, 22] 이다. K 로 나누면
%	 3    2   0   2   0   1   2   가 된다.
K 의 순서쌍은 (1, 3),  ( 2, 2 ) 로 나눌 수 있다.
최대의 개수를 구하는 것이기에 순서쌍에서 제일 큰 값을 각각 구한다면
Ex) 
나머지로 생각하자.
나머지가 1 과 3 이 되는 값들 중 가장 많은 갯수를 ResultCount 에 포함한다.
왜냐하면, 1, 3 은 동급으로 3 은 나머지가 1이 아닌 수만 아니면 어느 수를 더해도 부분집합을 성립한다.
다른 순서쌍도 마찬가지다.

예외로 ( 2, 2 ) 와 같이 같은 숫자가 나온다면, 이런 숫자가 나온다는 것은 K = "짝수" 일 때이다.
만약 K = "짝수" 이면,
2 + @ 는 무슨 수를 더하더라도 %K != 0 이 된다. ( @ 가 2가 아닐때 )
하지만 나머지가 2 인 수가 두개 이상 들어온다면, 2 + 2 % K == 0 이기 때문에 부분집합을 성립하지 않는다.
그래서 ResultCount++ 해준다.

0 + @ 는 무슨 수를 더하더라도 %K != 0 이 된다. ( @ 가 0 이 아닐 때 )
허나, 0 + 0 일 때에는 그 부분집합은 성립하지 않게 된다.
따라서 0 이 있다면, ResultCount++ 개를 해준다.

다시 정리하면,

1. K 의 순서쌍을 구한다.
2. S 집합을 K 로 나눈 값들을 구한다.
3. 총 ResultCount 를 계산한다.
