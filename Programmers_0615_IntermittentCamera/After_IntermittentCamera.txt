// * 문제
- 문제이름 : 단속 카메라

고속도로를 이동하는 모든 차량이 고속도로를 이용하면서 단속용 카메라를 한 번은 만나도록 카메라를 설치하려고 합니다.

고속도로를 이동하는 차량의 경로 routes가 매개변수로 주어질 때,
모든 차량이 한 번은 단속용 카메라를 만나도록 하려면 최소 몇 대의 카메라를 설치해야 하는지를
return 하도록 solution 함수를 완성하세요.

제한사항

차량의 대수는 1대 이상 10,000대 이하입니다.
routes에는 차량의 이동 경로가 포함되어 있으며 routes[i][0]에는 i번째 차량이 고속도로에 진입한 지점,
routes[i][1]에는 i번째 차량이 고속도로에서 나간 지점이 적혀 있습니다.
차량의 진입/진출 지점에 카메라가 설치되어 있어도 카메라를 만난것으로 간주합니다.
차량의 진입 지점, 진출 지점은 -30,000 이상 30,000 이하입니다.
입출력 예

routes	return
[[-20,15], [-14,-5], [-18,-13], [-5,-3]]	2
입출력 예 설명

-5 지점에 카메라를 설치하면 두 번째, 네 번째 차량이 카메라를 만납니다.

-15 지점에 카메라를 설치하면 첫 번째, 세 번째 차량이 카메라를 만납니다.

/////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
// * 해결 방법

1. 정렬을 사용해 기준을 잡는다.
그냥 구하려고 시도하면 매우 복잡해진다.

2. 만약 routes[0] 이 기준(temp) 보다 크다면 범위가 겹쳐지지 않으므로 카메라가 한 대 더 필요하다.

/////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
// * 시행착오

정렬을 사용하지 않고 쓰면 매우 복잡해진다.
처음엔 두 범위의 거리 순으로 우선순위를 사용해서 작은 순부터 점차 계산해가는 방식으로 정의했는데,
그렇게 된다해도 결국엔 두 범위를 각각 따로 계산해야된다.
정렬을 통해서 처음 범위를 맞춰놓고 routes[1] 을 이용해서 각각이 계산해가는 방식을 사용했다.
